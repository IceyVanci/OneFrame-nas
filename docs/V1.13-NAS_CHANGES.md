## 二、首页缩略图随机选取

### 问题描述

1. HTML 中 img src 指向不存在的根目录文件（如 `TypeA-sample_compressed.jpg`），实际文件在 `Sample/` 子目录中且使用 `.jpeg` 扩展名；
2. 缩略图选择器模块（`thumbnail-selector.js`）尚未移植。

### 修复内容

新建文件：`src/renderer/js/thumbnail-selector.js`

基于原始项目 `thumbnail-selector.js` 移植，适配 NAS 纯 Web 环境：

- 移除 `window.electronAPI?.getSampleFiles` Electron IPC 分支；
- 仅使用浏览器原生 API 探测文件存在性；
- `isValidSampleFilename()` 正则兼容 `.jpeg`、`.jpg`、`.png`、`.webp` 扩展名；
- `collectCandidatesForStyle()` 先尝试 `.jpeg` 扩展名，再回退尝试 `.jpg`；
- Fisher-Yates 洗牌算法保证均匀随机；
- 全局 imageId 去重：不同样式尽量不重复使用同一张图片。

修改文件：`src/renderer/index.html`

## 三、刷新页面时缩略图闪烁两次

### 问题描述

原实现使用 `new Image()` 对象探测文件存在性，每次探测都会发出完整图片加载请求。探测过程中浏览器先渲染 HTML 中的默认 img src（第一次显示），异步探测完成后 `initHomepageThumbnails` 将 img.src 替换为随机选中的图片（第二次加载），导致用户看到图片闪烁/替换两次。

### 修复内容

修改文件：`src/renderer/js/thumbnail-selector.js`

将 `checkFileExists()` 从 Image 对象探测改为 `fetch HEAD` 请求：

```javascript
async function checkFileExists(url) {
  try {
    const resp = await fetch(url, { method: 'HEAD' });
    return resp.ok;
  } catch {
    return false;
  }
}
```

HEAD 请求只获取 HTTP 头信息，不加载图片体，速度极快且不产生视觉副作用。

## 四、部分样式导出遗失焦距单位 mm

### 问题描述

`src/renderer/js/exif.js` 中的 `getFocalLength()` 函数返回原始 EXIF 值。`FocalLengthIn35mmFilm` 可能返回纯数字（如 `50`，无 "mm" 后缀），而 `FocalLength` 经过 `exifKeyFormatter` 处理后可能带 "mm"（如 `50mm`）。

部分导出文件（如 Type A、B、C、D export）直接使用 `settings.focalLength` 而不做 `.replace(/mm$/i, '')` + 重新添加 "mm" 的处理，导致当值为纯数字时导出图片上缺少 "mm" 单位。

### 修复内容

修改文件：`src/renderer/js/exif.js`

在 `getFocalLength()` 中统一确保返回值带 "mm" 后缀：

```javascript
export function getFocalLength(exif) {
  let focal;
  if (exif['FocalLengthIn35mmFilm']) {
    focal = exif.FocalLengthIn35mmFilm;
  } else if (exif.FocalLength) {
    focal = exif.FocalLength;
  }
  if (!focal) return null;
  const str = String(focal).trim();
  return str.endsWith('mm') ? str : `${str}mm`;
}
```

已做 `.replace(/mm$/i, '')` + 重新添加 "mm" 的样式（Type E、F、G、H、I、J、K、L、M）不受影响，因为它们会先去掉再加回。
