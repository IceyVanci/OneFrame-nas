## 二、Type E 图片拖动边界修复

### 问题描述

Type E 顶部 1:1 正方形图片区域支持拖动选择裁剪位置，但旧实现存在坐标系混用问题：

- 预览端最大拖动范围使用原始图片像素与预览 CSS 像素直接相减；
- `state.imageOffset` 与 CSS `object-position` 的数学模型不统一；
- 拖动后可能出现预览位置、内部偏移状态和导出裁剪位置不一致。

### 修复内容

修改文件：`src/renderer/js/styles/type-E-preview.js`

新增内部辅助函数：

- `clamp(value, min, max)`：统一数值钳制；
- `getRenderedImageMetrics()`：按 `object-fit: cover` 规则计算图片渲染后的预览 CSS 像素尺寸；
- `clampImageOffset(offset)`：将偏移限制到当前图片可裁剪范围；
- `applyImageOffset()`：统一钳制 `state.imageOffset` 并同步写入 DOM 的 `object-position`。

关键规则：

- 纵向图片：宽度填满正方形，高度按比例溢出，可上下拖动；
- 横向图片：高度填满正方形，宽度按比例溢出，可左右拖动；
- 方形图片：无有效溢出，不显示拖动提示。

`calculateObjectPosition()` 改为将有效偏移映射到 `0%~100%`：

```javascript
const xPercent = maxOffsetX > 0 ? 50 + (x / maxOffsetX) * 50 : 50;
const yPercent = maxOffsetY > 0 ? 50 + (y / maxOffsetY) * 50 : 50;
```

这样 `-maxOffset` 对应 `0%`，`0` 对应 `50%`，`+maxOffset` 对应 `100%`。

### 提示文字恢复

移除临时禁用说明：

- `↑↓ 纵向图像暂时无法使用 ↓↑`
- `←→ 拖动功能暂时失效 →←`

恢复为正常拖动提示：

- `↑↓ 拖动选择区域 ↓↑`
- `←→ 拖动选择区域 →←`

## 三、Type E 导出偏移方向修复

### 问题描述

Type E 拖动后导出的裁剪区域可能与预览方向相反。

### 修复内容

修改文件：`src/renderer/js/styles/type-E-export.js`

将导出 Canvas 中图片绘制偏移方向从加法改为减法：

```javascript
const drawOffsetX = baseOffsetX - imageOffset.x * squareSize;
const drawOffsetY = baseOffsetY - imageOffset.y * squareSize;
```

这样导出端与 CSS `object-position` 的预览语义保持一致。

## 四、重选图片 EXIF 刷新与 Type E 偏移重置

### 问题描述

重新选择图片时存在以下风险：

- 浏览器路径中重新选择图片未 `await loadImageWithExif()`；
- 快速连续重选图片时，较早一次异步 EXIF 结果可能覆盖较晚选择；
- Type E 新图片可能沿用上一张图片的拖动偏移；
- 兼容保留的 Electron 分支中，EXIF 表单更新与图片 `load` 事件存在不必要耦合；
- Electron 无 EXIF 时的文件修改时间回退值可能被 `resetForm()` 清空。

### 修复内容

修改文件：`src/renderer/js/app.js`

新增：

- `let imageLoadSequence = 0;`
- `formatDateTimeForInput(value)`

调整：

- `loadImageWithExif(file)`：增加加载序号，防止旧异步结果覆盖新图片；Type E 加载新图片时调用 `typeEPreview.resetImageOffset()`；
- `loadImageInElectron(imagePath)`：保留兼容分支但修复时序问题，使用局部 `fallbackDateTimeValue` 保存文件修改时间回退值；
- `updateExifDisplay()`：统一使用 `formatDateTimeForInput()` 处理 `DateTimeOriginal`；
- 浏览器路径 `btnReselect` 回调改为 async，并 `await loadImageWithExif()`。

保留行为：

- Type B 横向图片限制逻辑不变；
- Type F / Type J 的 `Make + Model` 机型显示逻辑不变；
- Type M 图框尺寸缓存清理逻辑保留；
- 非 Type E 样式不传递 `imageOffset`，导出协议不变。

## 六、建议验证

### Type E 拖动验证

- 选择纵向图片，上下拖动到边界，确认无露白且提示文字正常；
- 选择横向图片，左右拖动到边界，确认无露白且提示文字正常；
- 选择方形图片，确认不显示可拖动提示或拖动不产生异常偏移。

### Type E 导出验证

- 纵向图片分别拖到顶部、中间、底部后导出；
- 横向图片分别拖到左侧、中间、右侧后导出；
- 确认导出裁剪区域与预览一致，不再反向。

### 重选图片验证

- 选择图片 A 后确认 EXIF 字段填充；
- 点击“重新选择”选择图片 B，确认字段刷新为图片 B；
- Type E 中拖动图片 A 后重选图片 B，确认图片 B 初始居中且导出不沿用旧偏移；
- 快速连续重选多张图片，确认最终字段对应最后一次选择。

### 跨样式回归

- Type B 横向图片限制仍有效；
- Type F / Type J 的机型显示逻辑仍正常；
- 其他样式重选图片后预览和动态背景仍正常刷新。
