## 一、Type I 极简叠加文字样式（移植自原始项目 v1.0.7）

### 设计说明

新增 Type I 样式（极简叠加文字），基于 Type H（全画幅叠加文字）进一步简化。

**设计目标：** 最大化照片展示面积，仅保留品牌 Logo 和署名，适合纯净的照片展示场景。

**与 Type H 的关键差异：**

| 特性 | Type H | Type I |
|------|--------|--------|
| Logo 位置 | 底部居中 | 顶部居中 |
| 底部内容 | 日期 + 参数 + 机型 + 署名 | 仅署名 |
| 编辑面板 | 隐藏边框/比例/开关，显示 Logo + 文字颜色 | 额外隐藏设备型号/拍摄参数/时间 |
| 默认署名 | 用户手动输入 | 自动填入 "OneFrame" |
| 纵向图片 | 无特殊处理 | 底部字体增大 50% |

### Type I 技术细节

**预览端 `calcSize()`：**
```javascript
return { squareSize: naturalWidth, canvasHeight: naturalHeight };
```

**纵向图片字号增大：**
```javascript
const isPortrait = state.img && state.img.naturalHeight > state.img.naturalWidth;
if (isPortrait) {
  baseFontSize = Math.round(baseFontSize * 1.5);
  state.borderContent.style.fontSize = `${baseFontSize}px`;
}
```

**字号累乘修复：** 每次 `updateContentPreview` 从 `frameWrapper.style.width` 重新计算基准字号，避免读取已被修改的 `borderContent.fontSize` 导致累乘。

## 二、Type J 署名+三栏参数行样式（移植自原始项目 v1.0.7）

### 设计说明

新增 Type J 样式（署名+三栏参数行），基于 Type H 调整布局。

**设计目标：** 不显示 Logo，署名移到原 Logo 位置，参数行重新布局为三栏（左机型/中参数/右时间）。

**与 Type H 的关键差异：**

| 特性 | Type H | Type J |
|------|--------|--------|
| Logo | 底部居中显示 | 不显示 |
| 署名位置 | 底部第三行 | 原 Logo 位置（底部第一行） |
| 署名字号 | 小于参数（0.8571em） | 与参数相同 |
| 参数行 | 单行居中（日期 \| 参数 \| 机型） | 三栏布局（左机型/中参数/右时间） |
| 机型名称 | 不含厂商 | 含厂商（如 "Sony A7M4"） |
| 编辑面板 | 隐藏边框/比例/开关，显示 Logo + 文字颜色 | 额外隐藏 Logo 区域和设备型号输入 |

### 新建文件

### Type J 机型名称含厂商

Type J 机型名称自动包含厂商前缀（与 Type F 相同逻辑）：
```javascript
if (currentStyle === 'type-f' || currentStyle === 'type-j') {
  const makeName = make ? getMakeName(make) : '';
  customModel.value = makeName ? `${makeName} ${modelName}` : modelName;
}
```

### Type J 导出定位

导出使用底部锚定 `bottom: 3%` 定位（与 CSS `.type-j-bottom { bottom: 3% }` 对齐），从底部向上排列：参数行在下，署名在上。
