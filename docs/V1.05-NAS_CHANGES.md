## 一、Type F 纵向图片自适应

### 修改说明

为 Type F（画中画风格）增加纵向图片自适应逻辑：当导入图片为纵向（高度 > 宽度）时，图框上下的白色部分减少一半，照片区域相应增大。

- **当前横向图片布局**：顶部 5% 白色留白 + 中部 80% 照片展示区 + 底部 15% 文字信息区
- **纵向图片新布局**：顶部 2.5% 白色留白 + 中部 90% 照片展示区 + 底部 7.5% 文字信息区

### 修改文件

| 文件 | 变更类型 | 说明 |
|------|----------|------|
| `src/renderer/js/styles/type-f-preview.js` | 修改 | 预览模块：`calcSize()` 纵向图片使用 0.9 计算画布高度；`updatePreview()` 纵向图片时覆盖 CSS 默认百分比值 |
| `src/renderer/js/styles/type-f-export.js` | 修改 | 导出模块：`renderImage()` 纵向图片使用 0.9 计算画布高度，调整照片区域参数；`drawBorderContent()` 接收纵向标记，调整文字区域比例 |
| `src/renderer/js/app.js` | 修改 | 主逻辑：Type F 分支画布高度计算改为自适应比例（纵向 0.9，横向 0.8） |

### 修改详情

#### 1. `type-f-preview.js` — `calcSize()` 函数

```javascript
// 新增纵向检测
const isPortrait = naturalHeight > naturalWidth;
const heightRatio = isPortrait ? 0.9 : 0.8;
const canvasHeight = Math.round(naturalHeight / heightRatio);
```

返回值增加 `isPortrait` 字段。

#### 2. `type-f-preview.js` — `updatePreview()` 函数

```javascript
// 纵向图片：覆盖 CSS 默认的 top:5% height:80% 为 top:2.5% height:90%
const isPortrait = imgDimensions.naturalHeight > imgDimensions.naturalWidth;
if (isPortrait) {
  state.img.style.top = '2.5%';
  state.img.style.height = '90%';
}

// 同步设置文字区域高度（纵向 7.5%，横向由 CSS 默认 15%）
if (state.borderContent) {
  state.borderContent.style.height = isPortrait ? '7.5%' : '';
}
```

#### 3. `type-f-export.js` — `renderImage()` 函数

```javascript
// 纵向图片画布高度计算
const isPortrait = img.naturalHeight > img.naturalWidth;
const heightRatio = isPortrait ? 0.9 : 0.8;
const canvasHeight = Math.round(img.naturalHeight / heightRatio);

// 照片区域参数调整
const photoY = canvasHeight * (isPortrait ? 0.025 : 0.05);
const photoHeight = canvasHeight * (isPortrait ? 0.90 : 0.80);
```

#### 4. `type-f-export.js` — `drawBorderContent()` 函数

签名增加 `isPortrait` 参数：
```javascript
async function drawBorderContent(ctx, canvasWidth, canvasHeight, settings, fonts, isPortrait = false)
```

文字区域比例调整：
```javascript
const textRatio = isPortrait ? 0.075 : 0.15;
const textAreaHeight = canvasHeight * textRatio;
const textAreaTop = canvasHeight * (1 - textRatio);
```

#### 5. `app.js` — `updateBorder()` Type F 分支

```javascript
// 画布原始尺寸（基于图片，不受窗口影响）
// 纵向图片照片占 90%（白色区域减半），横向图片照片占 80%（默认）
const canvasW = userImage.naturalWidth;
const isPortraitF = userImage.naturalHeight > userImage.naturalWidth;
const heightRatioF = isPortraitF ? 0.9 : 0.8;
const canvasH = Math.round(userImage.naturalHeight / heightRatioF);
```

### 测试验证

1. 横向图片：布局不变（顶部 5%，照片 80%，底部 15%）
2. 纵向图片：顶部白色区域明显更窄（2.5%），照片区域更大（90%），底部文字区更窄（7.5%）
3. 窗口缩放后，纵向图片的图框比例保持正确
4. 导出结果与预览一致
5. 其他样式不受影响

### 备注

- 本次修改仅影响 Type F 样式，不影响其他样式
- 所有条件判断基于 `naturalHeight > naturalWidth`，确保横向图片行为不变
- 修改同时覆盖预览和导出两个路径，保持一致性

## 二、Type E 文字缩放

### 修改说明

为 Type E（3:2 纵向）添加文字和 Logo 随画布大小缩放的机制，修复预览和导出两端的文字大小问题。

**问题 1（预览端）**：Type E 预览使用 CSS 固定像素字号（月份 48px、年份 24px、参数 18px），当 frameWrapper 缩放到预览区域时，文字不会跟随缩放，导致大窗口时文字相对偏小。

**问题 2（导出端）**：导出模块的缩放基准依赖预览窗口的 `previewSquareSize`，不同窗口大小会导致导出文字大小不一致。

**解决方案**：
- CSS 字号从 `px` 改为 `em` 单位（相对于基准字号）
- 预览端基准字号动态计算：`24 * squareSize / 480`（基准 480px 对应 24px）
- 导出端缩放基准固定为 480px：`canvasWidth / 480`

### 修改文件

| 文件 | 变更类型 | 说明 |
|------|----------|------|
| `src/renderer/css/type-e.css` | 修改 | font-size 从 px 改为 em（month: 2em, year: 1em, param: 0.75em），margin-bottom 从 8px 改为 0.333em |
| `src/renderer/js/styles/type-e-preview.js` | 修改 | `updateFrameWrapper()` 动态基准字号；`handleLogoLoad()` 从 borderContent 读取动态字号 |
| `src/renderer/js/styles/type-e-export.js` | 修改 | `drawBorderContentTypeE()` 缩放基准从 `previewSquareSize` 改为固定 480 |

### 修改详情

#### 1. `type-e.css` — font-size 改为 em

```css
/* 修改前 */
.type-e-month  { font-size: 48px; margin-bottom: 8px; }
.type-e-year   { font-size: 24px; }
.type-e-param-line { font-size: 18px; }

/* 修改后 */
.type-e-month  { font-size: 2em;    margin-bottom: 0.333em; } /* 48/24=2, 8/24≈0.333 */
.type-e-year   { font-size: 1em;    } /* 24/24=1 */
.type-e-param-line { font-size: 0.75em; } /* 18/24=0.75 */
```

#### 2. `type-e-preview.js` — `updateFrameWrapper()`

```javascript
// 动态设置基准字号，CSS 中的 em 单位会相对于此值缩放
// 基准 480px 对应 24px，随 squareSize 等比缩放
if (state.borderContent) {
  const baseFontSize = Math.max(8, Math.round(24 * squareSize / 480));
  state.borderContent.style.fontSize = `${baseFontSize}px`;
}
```

#### 3. `type-e-preview.js` — `handleLogoLoad()`

```javascript
// 从 borderContent 读取动态基准字号（替代硬编码 24）
const baseFontSize = state.borderContent
  ? parseFloat(getComputedStyle(state.borderContent).fontSize)
  : 24;
const yearFontSize = baseFontSize;
```

#### 4. `type-e-export.js` — `drawBorderContentTypeE()`

```javascript
// 修改前：依赖预览窗口大小，不同窗口导出字号不同
const scale = previewSquareSize > 0 ? canvasWidth / previewSquareSize : 1;

// 修改后：固定基准 480px，与 TYPE_E_STYLES 常量一致
const scale = canvasWidth / 480;
```

### 缩放机制说明

| 组件 | 字号计算 | 示例（squareSize=700, canvasWidth=3000） |
|------|---------|------|
| 预览 - 基准字号 | `24 × squareSize / 480` | `24 × 700 / 480 = 35px` |
| 预览 - 月份 | `baseFontSize × 2em` | `35 × 2 = 70px` |
| 预览 - 年份 | `baseFontSize × 1em` | `35 × 1 = 35px` |
| 预览 - 参数 | `baseFontSize × 0.75em` | `35 × 0.75 = 26px` |
| 导出 - 缩放比例 | `canvasWidth / 480` | `3000 / 480 = 6.25` |
| 导出 - 年份 | `24 × scale` | `24 × 6.25 = 150px` |

预览和导出的文字/画布比例完全一致，不受窗口大小影响。
