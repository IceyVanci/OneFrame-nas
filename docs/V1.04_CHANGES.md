**日期**：2026-06-19

## 修复 Type F 预览缩放问题

### 问题描述

Type F（画中画样式）的预览界面存在多个缩放相关问题：

1. 窗口大小变化后图框与图片相对位置/比例关系错乱
2. 非初始窗口大小下导入图片时图框比例错误
3. 横向窗口变小时图框不缩放（只减小横向宽度）
4. 纵向窗口变大时图框只增大纵向高度，横向不跟随

### 根因分析

两个独立问题：

#### 问题 1：双重缩放（calcSize + transform: scale）

`type-f-preview.js` 的 `calcSize()` 函数返回原始画布尺寸，但 `app.js` 的 `updateBorder()` Type F 分支对这些值应用了 `transform: scale()`。`transform: scale()` 只改变视觉大小不改变布局大小，与 CSS 百分比定位（`width:92%`, `height:80%`）和父容器 `overflow: hidden` 产生冲突。

#### 问题 2：CSS 选择器意外匹配

`type-a.css` 中 `.frame-wrapper:not(.type-b):not(.type-e)` 通用选择器匹配了 Type F，施加了 `max-width: 900px`、`width: 100%`、`max-height: 100%` 约束。

### 修复方案

| 文件 | 变更 |
|------|------|
| `src/renderer/css/type-f.css` | 添加 `max-width: none !important; max-height: none !important;` 覆盖继承约束 |
| `src/renderer/js/app.js` | Type F 分支去掉 `transform: scale()`，改为每次 resize 动态计算显示尺寸 `displayScale = min(availW/canvasW, availH/canvasH, 1)` |

### 修复原理

所有元素的缩放比例完全相同，以维持相对大小和位置不变：

1. 画布原始尺寸完全基于图片（`canvasW = naturalWidth`，`canvasH = naturalHeight / 0.8`）
2. 每次 resize 时计算 `displayScale`，得到 `displayW` 和 `displayH`
3. `updateFrameWrapper(displayW, displayH)` — 布局尺寸 = 显示尺寸
4. CSS 百分比定位（图片 92%×80%、文字区 15%）自动按显示尺寸计算
5. 字号 = `14 × displayW / 900`，跟随显示宽度
6. 无 transform，无双重缩放，所有元素同比例缩放
