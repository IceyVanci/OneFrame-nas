## 一、Type H 全画幅叠加文字样式（移植自原始项目 v1.0.6）

### 设计说明

新增 Type H 样式（全画幅叠加文字），基于 Type G 但移除白色边框，Logo 和文字直接叠加在照片内部底部。

**与 Type G 的关键差异：**

| 特性 | Type G | Type H |
|------|--------|--------|
| 画布大小 | 图片高度 ÷ 0.8（横向）/ 0.9（纵向） | 图片原始大小（无额外白色区域） |
| 照片区域 | 92%×80%，顶部 5% 留白 | 100%×100% 填满画布 |
| 纵向图片 | 白色区域减半 | 不做任何调整 |
| 文字/Logo | 叠加在底部白色区域 | 叠加在照片底部内部 |
| 文字颜色 | 黑色 #000000 | 白色 #ffffff（在照片上更易读） |
| 背景色 | 白色 #ffffff | 黑色 #000000（照片填满，背景不可见） |
| 编辑面板 | 隐藏开关，显示 Logo 选择 | 完全相同 + 显示文字颜色选择 |

### Type H 技术细节

**预览端 `calcSize()`：**
```javascript
// Type G: canvasHeight = naturalHeight / 0.8
// Type H: canvasHeight = naturalHeight（无除法）
return { squareSize: naturalWidth, canvasHeight: naturalHeight };
```

**导出端 `renderImage()`：**
```javascript
// Type G: canvasHeight = naturalHeight / 0.8，照片绘制为 92%×80%
// Type H: canvasHeight = naturalHeight，照片绘制为 100%×100%
```

**文字颜色：**
- Type H CSS 使用 `color: #ffffff`（白色，在照片上更易读）
- 导出端 `drawText()` 默认 color 为 `#ffffff`
- 预览端通过 inline style 覆盖 CSS 默认颜色

### Type H Logo 大小规则

与 Type G 相同的公式（相对于画布的比例）：
- Logo 最大高度 = 画布高度的 2.5%（即 `canvasHeight × 0.025`）
- 宽度：按原始宽高比自动计算

由于 Type H 的画布 = 图片原始大小，Logo 实际尺寸会比 Type G 略小。

### Type H 编辑面板特殊配置

与 Type G 完全相同：
- 隐藏边框颜色/高度/比例设置
- 隐藏所有显示开关（默认激活）
- 保留 Logo 选择区域
- 设备型号 placeholder 为"型号（如 A7M4）"
- **额外显示文字颜色选择器**（黑/灰/白）

## 二、文字颜色选择功能（Type H 专属）

### 背景

Type H（照片填满画布，文字叠加）的文字颜色默认为白色，但在浅色照片上可能看不清，因此需要用户手动选择黑/灰/白文字颜色。

### 实现

| 文件 | 变更类型 | 说明 |
|------|----------|------|
| `src/renderer/index.html` | 修改 | 添加 `#textColorSection`（默认 display:none），包含黑/灰/白三个预设按钮 |
| `src/renderer/js/app.js` | 修改 | `getDisplaySettings()` 和 `getEditSettings()` 添加 textColor 字段；`updateBorderContent()` 传递 textColor；添加 textColorPresets 点击事件 |
| `src/renderer/js/components/type-h-editor-panel.js` | 新建 | 显示 `#textColorSection`，默认激活白色按钮 |
| `src/renderer/js/styles/type-h-preview.js` | 新建 | `updateContentPreview()` 使用 `textColor` 参数，通过 inline style 覆盖颜色 |
| `src/renderer/js/styles/type-h-export.js` | 新建 | `drawBorderContent()` 使用 `settings.textColor || '#ffffff'` |

### 兼容性

- 其他样式（Type A-G）不受影响，`#textColorSection` 对它们保持 `display:none`
- `app.js` 和 `exporter.js` 无需额外修改，`textColor` 作为通用字段传递

## 三、Bug 修复

### 修复文字颜色切换预览不更新

**问题**：Type H 模式下点击文字颜色按钮，预览不更新。

**根因**：`app.js` 中通用边框颜色事件处理器使用 `.color-preset` 选择器，会匹配到文字颜色按钮（`.color-preset[data-text-color]`），导致：
1. 点击文字颜色按钮时，边框颜色处理器先触发，移除所有 `.color-preset` 的 active，设置 `borderColor.value = undefined`，调用 `updateBorder()`
2. 文字颜色处理器后触发，但状态已被覆盖

**修复**：将通用边框颜色事件处理器选择器从 `.color-preset` 改为 `.color-preset[data-color]`，与原始项目一致。

```javascript
// 修复前
document.querySelectorAll('.color-preset').forEach(btn => { ... });
// 修复后
document.querySelectorAll('.color-preset[data-color]').forEach(btn => { ... });
```

**影响范围**：仅影响边框颜色按钮的事件绑定，不影响其他模式的正常使用。
