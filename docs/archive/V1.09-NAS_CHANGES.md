# OneFrame NAS v1.09 阶段性修改记录

**日期**：2026-06-29

---

## 一、主界面图片墙重构

将主界面从单列纵向排列重构为铺满窗口的 3 列瀑布流图片墙，移除标签栏，所有 12 种样式同时显示。

- `.style-grid`：从 `display: grid` 改为 `column-count: 3`
- `.style-card`：添加 `break-inside: avoid` + `margin-bottom: 12px`
- `.app-header` 和 `.style-gallery`：移除 `max-width: 1000px` 限制
- 删除标签栏 `<div class="tab-bar">` 和 `data-category` 属性
- 修正 I/J/K/L 预览图引用（各自专用图片）

---

## 二、Type K/L 编辑面板改进

Type K 和 Type L 编辑面板现在显示设备型号输入框，方便用户手动修改机型名称。

---

## 三、文件名大写化（48 个文件）

将所有样式相关文件名中的样式字母从小写改为大写（如 `type-a.css` → `type-A.css`）。

**命名规范：**
- 文件名：大写（`type-A.css`、`type-A-preview.js`）
- CSS 类名：小写（`.type-a`）
- data-style 属性：小写（`data-style="type-a"`）
- JS 样式 ID：小写（`currentStyle === 'type-a'`）

---

## 四、NAS 移植修复（从原始项目 v1.09 同步后的适配）

### 4.1 index.html 本地资源修复

原始项目 v1.09 使用 CDN 链接，NAS 环境无外网，改为本地路径：

| 资源 | 原始项目（CDN） | NAS 版（本地） |
|------|----------------|---------------|
| Font Awesome | `cdnjs.cloudflare.com` | `assets/font-awesome/css/all.min.css` |
| opentype.js | `cdnjs.cloudflare.com` | `assets/opentype.min.js` |
| exifreader | 未引用 | `assets/exifreader.js` |

### 4.2 exporter.js 移除 Electron 专用代码

原始项目的 `exporter.js` 导入了 `readExifFromPath`（Electron 专用），NAS 版的 `exif-exporter.js` 没有此导出。移除该 import 和对应的 Electron 代码路径。

### 4.3 app.js 导出功能修复

原始项目 v1.09 的 `exportImageHandler` 只有 Electron 保存逻辑，缺少浏览器下载。添加 `if/else` 分支：
- Electron 环境：`window.electronAPI.saveBlob()`
- 浏览器环境：`<a download>` 自动下载

### 4.4 logo-utils.js 清理不存在的品牌

从 `logoList` 和 `logoSvgMap` 中删除没有 SVG 文件的品牌：
- 删除：Huawei、Insta360、OPPO、Samsung、xuzhou
- 保留：19 家有实际 SVG 文件的品牌

### 4.5 exif.js SUPPORTED_MAKES 清理

同步清理 `SUPPORTED_MAKES` 数组，移除不存在的品牌（华为、Insta360、OPPO、三星、徐州）。

### 4.6 app.js 拍摄时间正则修复

`updateExifDisplay()` 中的日期正则匹配格式不兼容：
- **修复前**：`/(\d{4}):(\d{2}):(\d{2}) (\d{2}):(\d{2})/`（仅匹配 `YYYY:MM:DD`）
- **修复后**：`/(\d{4})[-:](\d{2})[-:](\d{2}) (\d{2}):(\d{2})/`（同时匹配 `YYYY-MM-DD` 和 `YYYY:MM:DD`）
